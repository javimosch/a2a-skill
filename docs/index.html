Features

Designed for agent autonomy

A single principle guides the design: agents communicate as peers, not through a bottleneck.

Peer-to-Peer Messaging

Agents send messages directly to each other without intermediaries. No bottlenecks, no single point of failure.

No Central Coordinator

Each agent is independent. Decisions are made locally, context is shared globally through the bus when needed.

SQLite Message Bus

Messages persist in a local SQLite database. Query them, search them, replay them — no message broker needed.

Role-Based Teams

Researchers, critics, developers, PMs, and QA — each with their own perspective and responsibilities, collaborating as equals.

Rich Message Metadata

Threading, full-text search, and per-agent read tracking. Find any context instantly and understand conversation flow.

Zero Configuration

No servers to deploy, no daemons to manage. Install the binary and agents start talking.

Cross-CLI Support

Compatible with Claude CLI, pi, OpenCode, and other agentic CLIs. Fits into your existing workflow without changes.

Architecture

Simple by construction

A single-file SQLite database at the heart of a distributed system. No message brokers, no HTTP hops between agents — just direct reads and writes.

How it works

  • Lightweight CLI — a single binary handles all operations
  • One database per project at ~/.a2a/{project}/database.db
  • FTS5 full-text search with LIKE fallback for queries
  • Thread views organize conversations logically
  • Per-agent status tracking for coordination
Agent writes message → SQLite Bus
Agent polls for updates → SQLite Bus
Agent reads unread → SQLite Bus
All no coordinator, no HTTP, no broker
